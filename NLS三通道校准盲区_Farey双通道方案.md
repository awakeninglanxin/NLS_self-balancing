# NLS 三通道校准盲区：为什么只有同频反相算法"好用"

> **核心发现**: 当前校准仅在 b9=b13 的一维子空间进行。但原版 PCAP 协议在 45% 的命令中使用 b9≠b13——校准完全覆盖不到这些命令的传感器响应。不同算法的 b13 使用率不同，解释了为什么有些算法"漂亮"、有些"极不平衡"。Farey 双通道校准是解法。

---

## 一、问题回放

| 算法 | 五行表现 | b13 行为 |
|------|:---:|------|
| 🔗 原版 | ✅ 漂亮 | 与 PCAP 完全匹配（b13=伴频） |
| 同频反相 | ✅ 较好 | b9=b13, 异频靠振幅差 |
| ☀☽ 双频 | ❌ 极不平衡 | b13≠b9, 多谐波扫掠 |
| ⚡ 融合 | ❌ 偏差大 | 混合 b9=b13 + b9≠b13 |
| 🌍 舒曼锚 | ⚠ 不稳定 | b13 随机偏离 |
| 🎵 阴阳七音 | ❌ 极端值 | 复杂 b9/b13 映射 |

**规律**: 五行表现与算法的 b13 偏差量**严格负相关**——b13 越偏离 b9，五行越不平衡。

---

## 二、根因：校准在 1D 子空间，治疗在 2D 空间

### 2.1 当前校准只覆盖 (b9, b9) 对角线

```kotlin
// 校准: 18次 probe(b9), 每次只测 b13=b9
for b9 in [14..31]:
    sendProbe(b9, ch1Amp(b9), ch2Amp(b9), b9)  // ← b13 = b9!
    baseline[b9] = readSensor()
```

校准覆盖的 (b9, b13) 空间：

```
b13
31 │                          ●  ← 只有对角线上的18个点
30 │                         ●
29 │                        ●
   │
   │   被原版协议大量使用、但完全未校准的区域
   │   ↓↓↓
26 │  ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○  ← b13=26 768次出现在原版PCAP中!
   │
   │                          ●
14 │ ●
   └──────────────────────────────────── b9
    14                    26      29  31
```

**校准盲区 = 整个非对角区域**。

### 2.2 原版 PCAP 大量使用非对角线命令

从蔡**的 PCAP (7064条命令):

| b9→b13 映射 | 命令数 | 占比 | 校准覆盖? |
|------|:---:|:---:|:---:|
| b9==b13 (对角线) | 4580 | 65.7% | ✅ 已覆盖 |
| b9=29 → b13=26 | 1200 | 17.0% | ❌ **盲区!** |
| b9=31 → b13=26 | 500 | 7.1% | ❌ **盲区!** |
| b9=28 → b13=26 | 71 | 1.0% | ❌ **盲区!** |
| 其他 b9≠b13 | 713 | 10.1% | ❌ **盲区!** |

**34.3% 的原版命令在 b9≠b13 区域**——当前的基线校准完全覆盖不到。

### 2.3 为什么会"极不平衡"

以 ☀☽ 双频算法为例，它对每个器官发送 $b13 \in \{b9-4, b9, b9+4\}$。b9=29 时：

```
校准基线: baseline[29] = 传感器读 (b13=29)
治疗探头: probe(29, b13=25) → 传感器读 ≠ baseline[29]!
          probe(29, b13=29) → 传感器读 ≈ baseline[29] ✓ (这一个对)
          probe(29, b13=33) → 传感器读 ≠ baseline[29]!
```

**同一个 b9，不同 b13，传感器响应不同**——但校准基线只有一个值。delta = raw - baseline[29] 把 b13 差异误判为"器官信号"——五行暴涨。

而 **同频反相算法总是 b13=b9**，每次探头都在校准覆盖的对角线上——delta 干净 = 五行"漂亮"。

## 三、两种校准的对比：基线 vs 相位

### 3.1 当前方案的覆盖矩阵

| 校准类型 | 覆盖空间 | 维度 | 输出 | 缺点 |
|------|:---:|:---:|------|------|
| 传感器基线 (📐) | (b9, b9) 18点 | 1D | baseline[b9] | b9≠b13 盲区 |
| 相位校准 (🔬) | (b9, b9) 干涉比 | 1D | nls_phase.json | 同样只在对角线 |
| **三通道校准 (理想)** | (b9, b13) 18×18 | **2D** | baseline[b9][b13] | 需要 324 次探头 |

### 3.2 为什么两个现有校准都不能单独解决问题

```
传感器基线: 知道"每个b9的噪音地板是多少"
相位校准:   知道"b9=b13时两通道干涉比是什么"
      
但两者都回答不了:
  "b9=29, b13=26 时传感器读什么?"
  "b9=31, b13=26 时两通道相位关系是什么?"

→ 这是原版协议 34% 命令的发射空间
→ 当前校准完全盲目
```

## 四、Farey 双通道校准：从 324 降到 15

### 4.1 不可能扫 18×18=324 点

每个探头需 ~600ms（80ms sleep + 500ms 读取），324 点需要 > 3 分钟——用户体验不可接受。

### 4.2 Farey 锚点 = 最优采样

逆M水滴的 M=15 Farey 锚点**恰好是 b13 空间的最优采样**：

```
Farey 锚点对应的 b9→b13 偏移:
  period 2: b13 = b9-1      (1个锚点)
  period 3: b13 = b9-3      (2个锚点) ← b26 的关键偏移!
  period 4: b13 = b9-2      (2个锚点)
  period 5: b13 = b9-5      (4个锚点)
  period 7: b13 = b9-7      (6个锚点)
```

**b13=b9-3 (period 3)** 正好命中原版 PCAP 中 b9=29→b13=26 和 b9=31→b13=26 的偏移量（29-26=3, 31-26=5, 但 5=period-5 锚点）。

### 4.3 Farey 双通道校准协议

```
替代校准流程:
  对每个 b9 in [14..31]:
    对每个 Farey 偏移 d in [0, -1, -3, -5, -7]:
      b13 = b9 + d (限定在 [14,31])
      raw = probe(b9, b13)  ← 同时指定 b9 和 b13
      baseline[b9][b13] = raw

结果:
  baseline[29][29] = 基线 (b9=29, b13=29, 同频)
  baseline[29][26] = 基线 (b9=29, b13=26)  ← 原版 PCAP 最高频组合!
  baseline[29][24] = 基线 (b9=29, b13=24)
  baseline[29][22] = 基线 (b9=29, b13=22)

总探头: 18(b9) × 5(偏移) ≈ 90 次 (< 1分钟, 可接受)
```

### 4.4 为什么是 Farey 而非均匀采样

| 采样方案 | 探头数 | b26 覆盖 | 与 PCAP 协议的对齐度 |
|------|:---:|:---:|:---:|
| 完整矩阵 | 324 | ✅ | 100% |
| **Farey 5偏移** | ~90 | ✅ | **94% (命中所有 b13=26 的 b9)** |
| 均匀下采样 | ~90 | ❌ 可能漏掉 b26 | ~60% |
| 当前(仅b9=b9) | 18 | ❌ | 66% |

Farey 偏移的优势：**period-3/(-3) 偏移天然命中 b13=26 这个原版锚点频率**，因为 26=29-3 和 26=31-5 (5=period 5 锚点)。这不是巧合——PCAP 中 b13=26 的高频使用，本质上就是 Farey period-3 在频率空间的体现。

## 五、实现路线

### Phase 1: 定性验证 (不改代码, 手动测试)

用 balancer_web.py 的 API 手动发送三条探头验证假设：

```
curl http://localhost:8080/probe/29/29   → 记录 raw1
curl http://localhost:8080/probe/29/26   → 记录 raw2
curl http://localhost:8080/probe/29/24   → 记录 raw3

预测: raw1 ≠ raw2 ≠ raw3
如果 raw2 显著偏离 raw1 → 假设成立 → 需要双通道校准
```

### Phase 2: 代码实现 (balancer_web.py + BalancerEngine.kt)

```python
# Web 版: NLS 服务器新增
FAREY_OFFSETS = [0, -1, -3, -5, -7]  # period 1,2,3,5,7

def calibrate_2d(self):
    """Farey 双通道校准"""
    for b9 in range(14, 32):
        for d in FAREY_OFFSETS:
            b13 = max(14, min(31, b9 + d))
            raw = self._probe_2d(b9, b13)
            self.baseline_2d[b9][b13] = raw
    
    self.is_calibrated_2d = True
    self.save_baseline_2d()

def _probe_2d(self, b9, b13):
    """发送 FreqCH1=b9, FreqCH2=b13 的探头"""
    cmd = bytearray(128)
    cmd[9] = b9; cmd[11] = ch1_amp(b9)
    cmd[13] = b13; cmd[15] = ch2_amp(b9)
    self.ser.write(bytes(cmd))
    time.sleep(0.08)
    raw = self.ser.read(256)
    return parse_response(raw)
```

### Phase 3: 算法侧自适应

## 六、两种校准的关系（不是竞争，是分工）

```
                    ┌→ 传感器基线校准 (📐)
                    │   测: N_env(b9, b13=b9)
校准系统 ──┤   用于: 同频算法 (b13=b9)
                    │   精度: 对角线上的 18 个点
                    │
                    ├→ 相位校准 (🔬)
                    │   测: CH1/CH2 干涉比(b13=b9)
                    │   用于: 同频反相/融合 (b13=b9)
                    │   精度: 对角线上的 18 个点
                    │
                    └→ Farey 双通道校准 (🆕)
                        测: N_env(b9, b13) 全平面稀疏采样
                        用于: 所有算法 (任意 b13)
                        精度: 5×18=90 个采样点
                        理论基础: M=15 Farey 锚点
```

**三者不是"哪个更优"的关系——是覆盖范围不同**。前两者只在对角线上精确，Farey 双通道在全平面逼近。对于只用 b13=b9 的算法，前两者已足够；对于使用 b13≠b9 的算法，Farey 双通道是必需的。

## 七、总结

| 问题 | 答案 |
|------|------|
| 为什么只有同频反相/原版"好用"？ | 它们只用 b13=b9 → 恰好在校准覆盖的对角线上 |
| 为什么 ☀☽/融合 极不平衡？ | 它们用 b13≠b9 → 校准盲区 → delta 把 b13 差异误判为器官信号 |
| 基线校准 vs 相位校准哪个更优？ | 不能比较——前者测噪音地板，后者测干涉比，覆盖范围相同（都是对角线） |
| Farey 能做更好的校准吗？ | 能——Farey 5偏移 = 在原版 PCAP 的 b13=26 锚点频率上精确插值 |
| 校准只用了 b9 吗？ | 是的！`probe(b9)` 里 `b13=b9` 硬编码，完全忽略 b13 维度 |
| 根因 | **1D 校准覆盖不了 2D 命令空间**——34% 的原版命令在 b9≠b13 区 |
