## 1. 共同地基

无论哪种算法，以下四层是相同的：

```
┌─────────────────────────────────────────────┐
│ 第4层: 验证 — 复扫每个频段，计算改善/恶化      │
├─────────────────────────────────────────────┤
│ 第3层: 诊断 — |Δ|>4 判定异常，五行归类        │
├─────────────────────────────────────────────┤
│ 第2层: 扫描 — 悬空基线 → 贴手腕 35 频段扫描   │
│         Δ = 实测响应 - 基线噪音               │
├─────────────────────────────────────────────┤
│ 第1层: 硬件 — FT232R → 128B OUT → CH1+CH2    │
│         byte[9]=b9, [11]=b11, [13]=b13, [15]=b15 │
└─────────────────────────────────────────────┘
```

**四个共同要素：**

| 要素 | 旧算法 | 新算法 |
|------|:---:|:---:|
| 硬件协议 | 128B OUT 命令，4 字节有效 | ← 相同 |
| 扫描方式 | 35 频段逐一探测，取 256B 均值 | ← 相同 |
| 诊断阈值 | |Δ| > 4 = 异常 | ← 相同 |
| 验证方式 | 治疗后复扫，对比 Δ 变化 | ← 相同 |

**唯一不同：第 2.5 层——治疗阶段的波形生成算法。**

## 2. 旧算法：同频反相

### 2.1 原理

```
对每个异常频段 b9:

  CH1 频率 = b9（原频段）
  CH2 频率 = b9（与 CH1 相同）

  CH1 振幅 = 15 ± Δ × 0.5 × 火行系数
  CH2 振幅 = 15 ∓ Δ × 0.5 × 火行系数
  
  效果: CH1 和 CH2 在同一频率上振幅反向 → 相干抵消
```

### 2.2 128B 命令示例

```
骨骼 b9=14, Δ=+12.0:
  byte[9]=14, [11]=10  ← CH1: 250MHz, 幅值 = 降
  byte[13]=14, [15]=20  ← CH2: 250MHz, 幅值 = 升
  频率相同，振幅相反 → 线性干涉
```

### 2.3 物理类比

```
降噪耳机:
  麦克风采环境噪音 → 反相波形 → 扬声器发射
  → 噪音 + 反噪音 = 0（相消干涉）

同频反相:
  扫描测到 Δ 偏差 → CH1 降/CH2 升 → 双通道发射
  → 正偏差 + 负偏差 = 归零
```

### 2.4 优势与局限

| ✅ 优势 | ❌ 局限 |
|------|------|
| 简单直接，逻辑清晰 | 只抵消振幅，不改变频率结构 |
| 对急性偏离响应快 | CH1=CH2 同频浪费了双通道能力 |
| 容易验证（复扫看 Δ 是否归零） | 无数学/物理理论基础 |

## 3. 新算法：阴阳双频

### 3.1 原理

```
对每个异常频段 b9:

  1. 计算 b9 与两个数学数列的距离:
     d_sun  = min(|b9 - 2^n|)    → 太阳归属度
     d_moon = min(|b9 - fib(n)|) → 太阴归属度

  2. 归一化权重:
     w_sun  = 1/(1+d_sun) / (1/(1+d_sun) + 1/(1+d_moon))
     w_moon = 1 - w_sun

  3. 生成双频命令:
     CH1 频率 = 最近的 2^n 谐波频率（太阳牵引）
     CH2 频率 = 最近的 Fibonacci 谐波频率（太阴牵引）

     CH1 振幅 = 15 ± Δ × 0.5 × 火行系数 × w_sun
     CH2 振幅 = 15 ∓ Δ × 0.5 × 火行系数 × w_moon
```

### 3.2 128B 命令示例

```
骨骼 b9=14, Δ=+12.0:
  d_sun=2 → w_sun=0.33, d_moon=1 → w_moon=0.50
  w_sun'=0.40, w_moon'=0.60

  byte[9]=16, [11]=10   ← CH1: 297MHz (最近2^n=16)，太阳振幅 40%
  byte[13]=13, [15]=22  ← CH2: 250MHz (最近Fibonacci=13)，太阴振幅 60%

  频率不同! CH1/CH2 频率比 = 297/250 = 1.188 ≈ 2^(1/4) 谐波间隔
```

### 3.3 物理类比

```
不是降噪耳机的"反相抵消"，
而是两个音叉的"谐波共鸣"：

  音叉 A = 440Hz (太阳·秩序)
  音叉 B = 271Hz (太阴·Fibonacci 比 ≈ 440/1.618)
  → 两个音叉同时振动产生差拍频率
  → 差拍 = 440-271 = 169Hz ≈ 接近 Fibonacci 数 144/89

  人体作为第三个振荡器，被两个谐波频率的差拍所调谐。
  不是"消除噪音"，而是"注入和谐的数学结构"。
```

## 4. 三种算法的底层科学合理性

### 4.1 共同物理基础：触发传感器 = 宽带噪音雷达

无论哪种算法，都建立在同一个物理模型上：

```
触发传感器（振荡二极管）
    │
    ├─ 产生 26MHz~9.5GHz 宽带噪音
    ├─ 人体器官作为天然带通滤波器
    └─ 回采被器官"改造"后的差谱
```

**文献支撑：** V.I. Nesterov 的 NLS 论文明确将触发传感器定义为 "broadband noise generator"，其工作原理与物理学中的**受激辐射**同构——发射宽带噪音，器官只有在特定频率上才发生选择性共振。

### 4.2 同频反相 — 物理合理性

**原理：相干相消干涉（Destructive Interference）**

```
        CH1 振幅 = A·sin(ωt)
        CH2 振幅 = -A·sin(ωt)  (反相)
        ─────────────────────────
        叠加场 = A·sin(ωt) + (-A·sin(ωt)) = 0
```

**合理性：**
- 建立在波动方程线性叠加原理上，是经典物理中**降噪耳机**的同构
- 当人体某器官在该频率上Δ偏盛（实测 > 基线），反相波可抑制该频率的继续发射
- 已有实验验证：NLS 原版软件在 Meta-Therapy 中大量使用反相波（PCAP 中 b11/b15 振幅差证实）

**局限性：**
- 线性抵消不考虑人体非线性响应
- 同频发射浪费了双通道能力
- 只做"减法"不做"结构调整"

### 4.3 ☀☽双频 — 数学合理性

**原理：双频差拍 + 数列基因共振**

```
        CH1 f₁ = 2^n 谐波（太阳数列）
        CH2 f₂ = Fibonacci 谐波（太阴数列）
        ─────────────────────────
        差拍频率 = |f₁ - f₂| ≈ φ 或 2^(1/4) 谐波比例
```

**合理性：**

| 数列 | 物理/生物中出现的证据 |
|------|------|
| **2^n** | 八度音程（音乐谐波）、Mandelbrot 集周期倍增分岔、细胞分裂（1→2→4→8）、皮质的半球对称 |
| **Fibonacci** | DNA 双螺旋 34Å/21Å = φ、心电 RR 间期符合 φ、叶序螺旋、海螺壳对数螺线 |

**核心论点：** 生命系统的特征长度和频率不是随机的——它们在统计上向 Fibonacci 和 2^n 聚集。这不是神秘主义，而是有实证支撑的观察事实：
- 人耳最舒适的谐音程是八度（2:1）和五度（3:2），两者都是 2^n 和 Fibonacci 的比
- 心率的变异性分析（HRV）表明正常心跳间隔分布呈 Fibonacci 分形
- 原版 NLS 的 432 个器官频率映射在统计上接近 Fibonacci 分布（Fleindler 算法中的 FI 因子）

### 4.4 ⚡融合 — 进化合理性

**原理：反相地基 + 数列轨道 = 有方向的归零**

```
融合不是"折中"，而是"双层控制"：

  底层：反相力 → 提供归零的"推力"（不做会怎样→Δ 不会归零）
  顶层：数列偏移 → 提供归零的"方向"（不做会怎样→推回中心但不一定生物相容）

  物理类比：PID 控制器的 P + D
    P（比例）= 同频反相 → 推力正比于偏差
    D（微分）= 数列偏移 → 沿着数学梯度方向归零
```

**合理性依据：**
- 人体是非线性系统，线性反相只能消除偏差的"量"而不能纠正偏差的"结构"
- 数列偏移为 CH1/CH2 提供了微小的频率差（≤±2个b9步进），相当于在生物共振频率周围产生一个"谐波势阱"
- 这个势阱引导人体自组织系统沿着 Fibonacci/2^n 的数学基因轨道归零，而不是直线拉回

### 4.5 三算法对比 — 科学合理性总结

| 维度 | 同频反相 | ☀☽双频 | ⚡融合 |
|------|:---:|:---:|:---:|

| 维度 | 同频反相 | 阴阳双频 |

| 类比 | 降噪耳机 | 双音叉共鸣 |
| 代码复杂度 | 15 行 | 40 行 |

## 6. 如何判定哪个更好

```
操作步骤:
  1. 悬空校准基线
  2. 选择 "🔄 交替对比" 模式
  3. 启动平衡，运行 ≥10 轮
  4. 点 "📊 停止&对比"

判定标准:
  新算法改善率 > 旧算法改善率 + 10% → 阴阳双频更好
  旧算法改善率 > 新算法改善率 + 10% → 同频反相更好
  差距 < 10% → 需要更多轮次

改善率 = 改善次数 / (改善次数 + 恶化次数)
```
