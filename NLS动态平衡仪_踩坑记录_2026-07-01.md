# NLS 动态平衡仪 — 开发踩坑记录

> 2026-07-01 | Web仪表盘 + APK精简 + FTDI驱动折腾

---

## 坑1：Zadig 替换 FTDI 驱动（WinUSB）

**现象**: 用 Zadig 把 FT232R 的驱动从 VCP 换成 WinUSB，期望用 pyusb bulk 端点直驱传感器回读。

**根因**: FT232R 是 UART 桥接芯片，不暴露原生 USB bulk 端点。WinUSB 驱动挂上去后，芯片的 bulk IN/OUT 端点全部返回 `LIBUSB_ERROR_NOT_FOUND`。

**教训**: 
- FT232R ≠ 原生 USB 设备，它是串口转 USB 桥
- VCP 模式 (`pyserial`) 是正道，不用折腾 D2XX/WinUSB
- 如果真要 broadband noise + FFT，需用 FT60x 或原生 USB MCU

**恢复**: 设备管理器 → 卸载 libusb 驱动（勾选删除）→ 拔插 USB → Windows 自动装回 FTDI VCP

---

## 坑2：Zadig → libusb-win32 第二次踩

**现象**: COM4 在设备管理器里显示但 pyserial 打不开（FileNotFoundError）。

**根因**: Zadig 换了 USB 级驱动，FTDI bus driver (ftdibus.sys) 被踢掉。COM4 的 ftser2k.sys 子设备成了孤儿。

**教训**: 同一坑踩两遍 → 牢记「拔插 USB + 卸载驱动」是唯一恢复方式。

---

## 坑3：8080 端口多进程僵尸

**现象**: 网页打不开，反复 `taskkill //F //IM python.exe` 无效，`netstat` 仍显示多个 LISTEN。

**根因**: Git Bash 的 `taskkill` 在 WorkBuddy sandbox 中不可靠，Python 进程杀不掉。

**解决**: `PowerShell: Get-Process python | Stop-Process -Force`

---

## 坑4：connect() 阻塞 HTTP 服务启动

**现象**: `balancer_web.py` 启动后端口无监听，curl 超时。

**根因**: `main()` 中先调用 `b.connect()`（可能因 COM 端口占用而挂起），`HTTPServer` 还没 start 就卡死了。

**解决**: 
```python
server = HTTPServer(...)  # 先启动服务
threading.Thread(target=_bg_connect, daemon=True).start()  # 后台连
```

---

## 坑5：ctx.roundRect() → 马赛克图空白

**现象**: Canvas 马赛克图渲染后只有空框。

**根因**: `CanvasRenderingContext2D.roundRect()` 是 Chrome 99+ / Firefox 112+ 才支持的 API，WorkBuddy 内嵌浏览器版本更老。

**解决**: 手写圆角矩形（4个 `arcTo`），后因用户说频谱图+马赛克多余而直接删除。

---

## 坑6：JS 箭头函数 + raw string 中文 key → 五行雷达空白

**现象**: 雷达图完全不渲染。

**根因**: 
1. Python `r"""..."""` raw string 中的 Unicode 中文在旧浏览器 `===` 比较时失败
2. 箭头函数 `() => {}` 在旧 JS 引擎中可能报语法错误

**解决**: 
- 全改 `function(){}` 
- 五行匹配改用数字索引 `for(j=0;j<5;j++){if(wu===names[j])...}`

---

## 坑7：catch(function(){}) 吞所有错误

**现象**: 所有图表静默失败，无任何提示。

**根因**: `fetch.then(...).catch(function(){})` — 空 catch 块抹掉所有异常，图表绘制失败也不报。

**解决**: 加 `#dbg` 红色调试 div，每个图表独立 try/catch 并显示错误信息。

---

## 坑8：雷达图平均聚合隐藏信号

**现象**: 五行雷达五个顶点总缩在中心附近，打点变化不明显。

**根因**: 简单平均 `sum(deltas)/N` — 火行 13 个器官中 +8 和 -8 互相抵消，均值趋近 0。

**解决**: 改用 `sum(abs(deltas))/N` — 取绝对偏差的平均幅值，正负不再抵消。

---

## 坑9：频谱图+马赛克图过度设计

**现象**: 用户反馈两个图表多余。

**根因**: 功能堆叠心态 — 不先问用户需求就自作主张加图。

**教训**: 先 MVP 再迭代。五行雷达 + 列表已经足够。

---

## 坑10：扫描范围局限于 b9=14~31

**现象**: 按 NLS 原版软件只扫 18 频段。

**根因**: 昨天已验证硬件支持 b9=1~35（26MHz~9.5GHz），但代码沿用旧范围。

**解决**: `B9_RANGE = range(1, 36)` + 补充 17 个低频/超高频段的器官映射。

---

## 今日铁律总结

```
1. FT232R 就是串口 → 别碰 D2XX/WinUSB
2. 内嵌浏览器 = ES5 环境 → 别用箭头函数、roundRect、??、?. 等新语法
3. catch 块不写日志 = 自戳双目
4. Windows 杀进程用 PowerShell，别用 Git Bash
5. HTTP 服务先启再连外设，初始化放后台线程
6. 用户没要的功能别主动加（频谱图/马赛克）
7. 聚合信号用绝对幅值，不用带符号均值
```
