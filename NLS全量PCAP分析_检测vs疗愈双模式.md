# NLS 全量 PCAP 分析：单器官检测内嵌脉冲 vs 全身疗愈协议

> 扫描 33 份 PCAP，发现原版 NLS 有两种截然不同的 USB 命令模式。
> 这解释了为什么之前从 2 份全身疗愈 PCAP 提取的参数不能直接套用到单器官检测上。

---

## 一、数据全景

| 分类 | 数量 | 疗愈命令合计 |
|------|:--:|:--:|
| 全身疗愈 PCAP | 3 | 6,951 条 |
| 单器官/检测 PCAP | 30 | 826 条 |

**33 份 PCAP 全部含有疗愈命令**——没有纯检测 PCAP。每一份检测扫描里都嵌入了少量治疗脉冲。

---

## 二、核心发现：两种模式 = 两种协议

```
┌─────────────────────────────────────────────────────────┐
│  Mode A: 全身疗愈协议（nls_cmq2cure, zly_cure）         │
│  · 同频 46% — 算法骨架，反复验证                         │
│  · CH1 均值 72.1 — 中高功率                               │
│  · Burst 1.4 — 连续微调                                   │
│  · 这是原版的"正式疗愈"                                   │
├─────────────────────────────────────────────────────────┤
│  Mode B: 检测内嵌脉冲（25+ 份 per-organ PCAP）            │
│  · 同频仅 17% — 83% 是异频                               │
│  · CH1 均值 48.2 — 比正式疗愈低 24 点                     │
│  · Burst 1.1 — 几乎没有连续微调                           │
│  · 这是检测探针附带的"试探性治疗"                         │
└─────────────────────────────────────────────────────────┘
```

### 单器官 PCAP 的"肾治疗"真相

| PCAP | 疗愈条数 | b9=28 CH1 | 其实是 |
|------|:--:|:--:|------|
| `leftkidneybackview` | 3 | **172** | 左肾检测 + 附带脉冲 |
| `kidney3` | 10 | 23 | 肾第三次检测 |
| `kidneyonly` | 3 | 34 | 仅肾检测 |
| `肾手心扫第2次` | 4 | 16 | 手心扫肾 |
| `肾手心扫第3次` | 3 | 14 | 同上 |
| **全身: nls_cmq2cure** | **2700** | **126.2** | **正式肾疗愈** |
| **全身: zly_cure** | **4187** | **122.0** | **正式肾疗愈** |

即使在 `leftkidneybackview` 里 b9=28 的 CH1=172（最大功率），也只有 1 条命令——**这是一次检测探针，不是一轮治疗。**

---

## 三、这对我们的反推意味着什么

### 好消息：之前的协议是正确的

我们 54%+46% 的参数来自全身疗愈 PCAP，**这就是原版的正式治疗协议**。30 份检测 PCAP 的异频主导模式不会动摇它——它们是另一个操作模式。

### 新信息：单器官试探脉冲的规律

检测时的嵌入脉冲有一套独立规则：

```
检测模式脉冲特征:
  · 83% 异频（CH1≠CH2）
  · CH1 平均低功率 48（vs 治疗 72）
  · 单次发射（burst≈1.0-1.1）
  · 覆盖 b9 范围广（中位数 8.3 个不同频段）
```

这意味着原版检测时不是纯"读"——它在**同时做轻微的"试探性调理"**。有点像医生听诊时顺便按一下穴位。

## 五、下一步建议

### 如果目标是"逼近原版疗愈"

**不需要这些检测 PCAP。** `nls_cmq2cure` 和 `zly_cure` 已经提供了精准的疗愈协议。30 份检测 PCAP 是另一套协议，不是同一套。

### 如果目标是"超越原版"

这些检测 PCAP 揭示了一个原版未充分利用的信息——**每个器官在检测时原版发了什么试探脉冲**。这可以和我们的 `scan()` 结果做关联，找到"原版认为哪个频段对哪个器官最敏感"。

## 六、回答你的问题

> 蔡的肾被治了 1157 次，丽梦儿的肾被治了 1865 次——你能判断是因为之前有单肾 PCAP 吗？

**不能。** 1157/1865 是直接从全身疗愈 PCAP 里统计的 b9=28 出现次数。你的单肾 PCAP 最高只有 10 条疗愈命令（kidney3），远不足以解释这个数字。

单器官 PCAP 的价值在另一个维度——它告诉我们原版在检测阶段做了什么，不是治疗阶段。
