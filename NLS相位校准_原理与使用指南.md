# NLS 双通道相位校准 — 原理、方法与解读

> **适用设备**: NLS 动态平衡仪 (FT232R 双通道触发传感器) | **版本**: v7.0 | **日期**: 2026-07-03

---

## 一、为什么要做相位校准？

### 1.1 不是所有"反相"都是真的反相

NLS 手环通过 USB 发送 128 字节命令配置 CH1 和 CH2 两个射频通道。当两通道设为相同频率(b9)但不同振幅(b11≠b15)时，软件期望产生相消干涉。

**但硬件不一定按你的意图执行**：

- 如果 FT232R 的两路输出是**数字反极性**（DAC 输出符号翻转），两通道只是振幅叠加，没有真实的相消干涉
- 如果硬件链路上存在**频率相关的延迟**，某些频率恰好延迟半个周期（真反相），某些则完全不对

### 1.2 相位曲线是硬件指纹

相位关系**不随软件会话改变**——它是你手环的硬件特性：
- 同一台电脑、同一个 USB 口 → 相位曲线永久不变
- 换电脑、换 USB 口、换手环 → 需要重新校准

校准只需跑一次，存为 `nls_phase.json`，之后每次启动自动加载。

## 二、校准原理

### 2.1 测试方法

对每个 b9 频率（1-35），发三条命令，测量传感器响应：

```
测试 A: CH1 主导  (b11=15, b15=3)   → 记录 avg_A
测试 B: 同相同幅  (b11=15, b15=15)  → 记录 avg_B  
测试 C: CH2 主导  (b11=3,  b15=15)  → 记录 avg_C
```

### 2.2 干涉比计算

```python
干涉比 = avg_B / (avg_A + avg_C)
```

### 2.3 判定规则

| 干涉比 | 物理含义 | 等级 | 对疗愈的影响 |
|:---:|------|:---:|------|
| < 0.5 | 强相消干涉（接近真π反相） | **强相消** | 干涉放大效果 4×+, 可降功率 40% |
| 0.5 - 0.85 | 中等相消干涉 | **中相消** | 部分干涉增益, 降功率 40% |
| 0.85 - 1.0 | 弱相消干涉 | **弱相消** | 正常功率 |
| ≥ 1.0 | 无干涉 / 数字反极性 | **无/反相** | 正常功率, 无干涉增益 |

### 2.4 实测案例

我们测了自己的手环（COM4），三条代表性频段：

| b9 | 频率 | 干涉比 | 判定 |
|:--:|------|:--:|------|
| 20 | ~840 MHz | 0.00 | **近乎完美反相** — 在此频率相消干涉极强 |
| 14 | ~250 MHz | 0.48 | 强相消 |
| 28 | ~2.8 GHz | 1.04 | 数字反极性 — 无相位效应 |

**b9=20 是关键发现**：840 MHz 恰好是硬件链路上延迟半周期的频率。这意味着在这个频段的同频反相疗法具有最强的干涉增益。

## 三、如何执行校准

### 3.1 自动校准（推荐）

每次基线校准完成后，系统会自动检查 `nls_phase.json`：

```
文件已存在 → 直接加载, 不重新扫描 (0秒)
文件丢失   → 自动扫描 35 频段, 保存结果 (~16秒)
```

### 3.2 手动重新校准

点击 Web 界面的 `🔬 验相位` 按钮，或调用 API：

```bash
curl http://localhost:8080/verify_phase
```

### 3.3 前置条件

- 手环必须连接（COM4）
- **传感器悬空**（不要接触人体）——测的是硬件链路本身的相位关系，不需要组织信号

## 四、校准结果如何使用

### 4.1 输出文件

```json
// nls_phase.json
{
  "14": {"interfer": 0.48, "grade": "强相消"},
  "20": {"interfer": 0.00, "grade": "强相消"},
  "28": {"interfer": 1.04, "grade": "无/反相"},
  ...
}
```

### 4.2 在算法中自动生效

**同频反相** (`balance_legacy`) 和 **融合** (`balance_fusion`) 两种算法会自动查表：

```python
ph = self.phase_calib.get(str(b9), {})
if ph.get("grade") in ("强相消", "中相消"):
    ph_boost = 0.6  # 降低 40% 功率, 干涉已放大效果
else:
    ph_boost = 1.0  # 正常功率
```

### 4.3 强相消频段的价值

| 频段 | b9 | 对应的器官 | 干涉增益 |
|------|:--:|------|:--:|
| 840 MHz | 20 | 皮肤/皮毛 | ⭐⭐⭐ 完美反相 |
| 250 MHz | 14 | 高频/A | ⭐⭐ 强相消 |

如果你的手环在 b9=20 有完美反相，那么在这个频段**不需要很大的振幅差就能产生显著干涉效应**。`balance_legacy` 会自动利用这一特性。

## 五、FAQ

**Q: 换电脑后需要重新校准吗？**
A: 需要。USB 控制器的时序特性会影响相位。把 `nls_phase.json` 拷贝过去无效，建议在新环境重新跑一次。

**Q: 干涉比 > 1.0 是什么意思？**
A: 两通道非但不相消，反而产生了相长干涉（或纯粹的振幅线性叠加）。这通常意味着数字反极性。

**Q: 为什么不能只测 3 个 b9？**
A: 相位是**频率相关的**。b9=20 是完美反相，b9=28 却完全没有相位效应。只有全扫才能发现哪个频段对你的手环最重要。

**Q: 校准后 balancer_web.py 的疗愈效果会变化吗？**
A: 会——而且这正是目的。同频反相和融合算法会根据你手环的实际相位曲线自动调整功率，避免在不必要的高功率下浪费能量，同时最大化强相消频段的效果。
